# Security Policy

## Supported Version

项目处于早期阶段，仅维护 `main` 分支的最新提交。

## Reporting a Vulnerability

公开后请优先使用 GitHub **Private vulnerability reporting** 提交安全问题，不要在公开 Issue 中披露密钥、利用步骤或其他敏感信息。

如果私密报告入口尚未启用，请通过仓库所有者的 GitHub 主页请求私密联系方式；公开消息中只说明需要报告安全问题，不附带技术细节。

适用范围包括：

- 验证脚本或 GitHub Actions 可能执行不受信任代码
- Skill 指令可能导致越权、数据丢失或敏感信息泄露
- 模板或实现交接包可能意外包含凭据和个人信息

一般功能错误和文档问题请使用 Issue 模板报告。
