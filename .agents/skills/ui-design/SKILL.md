# UI Design

用于设计项目界面规范文档。

## When to Use

- 架构设计完成后，需要细化界面
- 用户说"设计 UI"、"画页面"、"界面布局"
- 需要定义组件规范和交互细节
- 需要为实现阶段提供可视化的界面参考

## NOT When to Use

- 只需要架构和功能设计，不需要具体 UI
- 项目是纯后端/CLI，没有图形界面
- 只需要数据模型和 API，不需要界面

## Input

| 来源 | 内容 |
|------|------|
| system-design.md | 模块划分、关键链路（确定需要哪些页面） |
| user-stories.md | 用户故事（确定用户视角的操作流程） |
| scope.md | 功能范围（确定 MVP 包含哪些界面） |
| 用户偏好 | 可选：风格倾向、目标用户画像 |

## Output Schema

| 文件 | 类型 | 说明 |
|------|------|------|
| `wiki/03-architecture/ui-design.md` | UI 设计规范 | 主文档，含风格定义 + 页面规范 + 组件规范 |
| `wiki/03-architecture/ui-design/assets/` | 资源目录 | ASCII 线框图源文件 |

## Minimum Viable Output

- UI 设计文档含：风格预设选择 + 至少 3 个核心页面的线框图 + 组件规范概要
- 每个页面有线框图（ASCII art）和简短说明
- 组件有关键状态说明

## Complete Output

- 完整 UI 设计文档，含：
  - 视觉基础（色板、字体、间距、圆角、阴影）
  - 风格预设应用结果
  - 所有页面的线框图（含导航流）
  - 组件规范（含所有状态）
  - 交互规范（动画、手势、加载态）
  - 无障碍要点
  - 情感化设计（空状态、鼓励文案）

## Dependencies

| 类型 | 说明 |
|------|------|
| 前置 | `architecture-decisions`（模块划分已定）、`requirements-spec`（用户故事已定） |
| 后置 | `delivery-planning`（任务拆解参考 UI 复杂度） |
| 依赖读取 | system-design.md、user-stories.md、scope.md |

## Procedure

### Phase 1: 理解上下文

1. 读取 `system-design.md` → 确定需要哪些页面
2. 读取 `user-stories.md` → 梳理用户操作流程
3. 读取 `scope.md` → 确定 MVP 包含哪些界面
4. 确定页面清单（首页/列表页/详情页/表单页/设置页等）

### Phase 2: 选择风格预设

展示 5 种风格预设供用户选择：

| 风格 | 关键词 | 适用场景 |
|------|--------|---------|
| **Modern Minimal** | 大量留白、细线、功能感 | 工具类、效率应用 |
| **Soft Elegant** | 柔和渐变、圆角卡片、温暖感 | 消费类、情感化产品 |
| **Neo-Brutalism** | 粗边框、高对比、无阴影 | 强调个性、年轻用户 |
| **Premium Dark** | 深色背景、金属光泽、精致动效 | 高端、专业工具 |
| **Playful Colorful** | 渐变、活泼、圆润 | 激励类、习惯追踪 |

**选择方式**：描述 5 种风格，用户选择（或说"让我来决定"则 AI 推荐）

### Phase 4: 设计页面结构

为每个页面创建 ASCII 线框图：

#### 页面模板类型

| 模板 | 用途 | 典型布局 |
|------|------|---------|
| `list-view` | 习惯列表、打卡记录 | 顶部导航 + 列表 + 底部 Tab |
| `stats-dashboard` | 统计图表、数据仪表盘 | 顶部筛选 + 图表区 + 数据卡片 |
| `form-wizard` | 创建习惯、设置提醒 | 步骤指示器 + 表单区 + 底部按钮 |
| `detail-view` | 单个习惯详情、历史记录 | 顶部详情 + 统计 + 历史列表 |
| `empty-state` | 无数据占位 | 居中图标 + 文案 + CTA 按钮 |
| `settings` | 设置页面 | 分组列表 + 开关/选项 |

### Phase 5: 定义组件规范

#### 按钮 (Button)

| 状态 | 说明 | 视觉表现 |
|------|------|---------|
| 默认 | 正常可点击 | 主色背景，白色文字 |
| 按压 | 手指按下 | 背景色加深 10% |
| 禁用 | 不可点击 | 50% 透明度，无交互 |
| 加载 | 等待中 | 显示 Spinner，文字变"处理中..." |

**变体**: Primary / Secondary / Ghost / Danger

#### 打卡按钮 (Checkin Button)

| 状态 | 说明 | 视觉表现 |
|------|------|---------|
| 未打卡 | 今天未完成 | 主色背景，大号按钮 |
| 已打卡 | 今天已完成 | 绿色背景 ✓ 图标 |
| 补打卡 | 可补签过去 | 次要样式，可选 |

#### 习惯卡片 (Habit Card)

| 状态 | 说明 |
|------|------|
| 正常 | 白色背景，显示习惯名 + 今日状态 |
| 按压 | 背景色变深 5% |
| 滑动 | 左滑显示编辑/删除 |
| 删除 | 软删除，不显示在列表 |

#### 图标 (Icon)

| 场景 | 尺寸 | 说明 |
|------|------|------|
| 导航栏 | 24×24px | 底部 Tab 和顶部操作按钮 |
| 列表项 | 20×20px | 列表项内的小图标 |
| 展示用 | 32×32px | 空状态、引导页用大图标 |

**命名规范**: 使用语义化名称（`ic_home`、`ic_settings`），而非 `icon_01`

#### 顶部导航栏 (Navbar)

```
┌─────────────────────────────────┐
│  ← 返回     页面标题      ⋮ 更多 │  ← 高度 44-56px
└─────────────────────────────────┘
```
| 元素 | 规范 |
|------|------|
| 高度 | 44px (iOS) / 56px (Android) |
| 标题 | 居中，H2 级别 |
| 左按钮 | 返回箭头或菜单 |
| 右按钮 | 操作菜单或设置 |

#### 底部标签栏 (Tabbar)

```
┌─────────────────────────────────┐
│  🏠 首页   📊 统计   ➕ 打卡   👤 我的 │  ← 高度 49-64px
└─────────────────────────────────┘
```
| 状态 | 视觉 |
|------|------|
| 选中 | 主色图标 + 标签文字 |
| 未选中 | 灰色图标 + 标签文字 |

#### 列表项 (List Item)

```
┌─────────────────────────────────┐
│  🟢 ●  习惯名称        连续15天  >  │  ← 高度 56-72px
└─────────────────────────────────┘
```
| 元素 | 规范 |
|------|------|
| 高度 | 56px (单行) / 72px (双行) |
| 左图标 | 20×20px 状态指示 |
| 主文字 | Body 16px |
| 辅助文字 | Caption 12px，次要色 |
| 右箭头 | Disclosure indicator |

#### 输入框 (Input/TextField)

| 状态 | 视觉 |
|------|------|
| 默认 | 浅灰边框，8px 圆角 |
| 聚焦 | 主色边框，深色边框 |
| 错误 | 红色边框 + 错误提示文字 |
| 禁用 | 50% 透明度，背景变灰 |

**结构**: `[标签] → [输入框] → [错误提示]`（垂直排列）

#### 开关 (Toggle)

| 状态 | 视觉 |
|------|------|
| 关闭 | 灰色轨道，48×24px |
| 打开 | 主色填充轨道 |
| 禁用 | 30% 透明度 |

#### 进度条 (Progress)

| 类型 | 用途 |
|------|------|
| 线性进度条 | 表单完成度、下载进度 |
| 环形进度 | 单指标统计展示 |
| 步骤指示器 | form-wizard 步骤导航 |

#### Toast / 提示消息

| 类型 | 用途 | 时长 |
|------|------|------|
| 成功 | 操作成功反馈 | 2s |
| 错误 | 操作失败提示 | 3s |
| 警告 | 风险提示 | 3s |
| 加载中 | 异步操作中 | 直到完成 |

#### 分割线 (Divider)

| 类型 | 用途 |
|------|------|
| 全宽分割线 | 区块间分隔 |
| 内缩分割线 | 列表项内部分隔，左侧留 16px |

#### 头像 (Avatar)

| 尺寸 | 用途 |
|------|------|
| 24×24px | 列表项内小头像 |
| 40×40px | 评论、消息内 |
| 64×64px | 个人中心大头像 |
| 80×80px | 初始加载页 |

**形状**: 圆形 (全圆角)

### Phase 6: 定义交互规范

#### 动画规范

| 交互 | 动画类型 | 时长 | 缓动 |
|------|---------|------|------|
| 页面切换 | 左右滑动 | 300ms | ease-in-out |
| 按钮按压 | 缩放 0.95 | 100ms | ease |
| 打卡成功 | 弹跳 + ✓ 缩放 | 400ms | bounce |
| 列表加载 | 淡入 | 200ms | ease |
| 弹窗出现 | 底部上滑 | 250ms | ease-out |

#### 手势规范

| 手势 | 场景 | 行为 |
|------|------|------|
| 下拉刷新 | 列表页 | 下拉触发刷新，显示 loading |
| 左滑操作 | 习惯列表 | 左滑显示快捷操作（编辑/删除） |
| 长按 | 习惯卡片 | 显示上下文菜单 |
| 点击反馈 | 所有可点击 | 100ms 内有明显按压反馈 |

#### 加载态

| 场景 | 方案 |
|------|------|
| 列表加载 | Skeleton 占位动画 |
| 图表加载 | 中心 Spinner |
| 按钮提交 | Spinner + 文字变化 |

### Phase 7: 无障碍要点

```markdown
## 无障碍要点

- [ ] 所有文本对比度 ≥ 4.5:1
- [ ] 触控区域 ≥ 44×44px
- [ ] 支持系统字体放大
- [ ] 颜色不作为唯一信息载体（如成功/失败除颜色外有图标/文字区分）
- [ ] 所有交互元素可聚焦
```


```markdown
### 打卡成功鼓励文案
```
